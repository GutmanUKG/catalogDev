Отчёт по доработкам markdown-модалки и корзины
Дата: 2026-02-25

Что было сделано:
1) В result_modifier.php доработан расчёт скидки:
   - Добавлен разбор условий склада не только в CONDITIONS_LIST, но и в ACTIONS_LIST.
   - Добавлена логика проверки соответствия склада для узлов CondCtrlProductOnWarehouse.
   - Для markdown-строк добавлен расчёт актуального количества в корзине по ключу строки (MARKDOWN_ROW_KEY), а не только по PRODUCT_ID.

2) В script.js доработана передача данных при добавлении markdown-товара в корзину:
   - В PROPS добавлены служебные свойства:
     MARKDOWN_STORE_ID, MARKDOWN_CATEGORY, MARKDOWN_ROW_KEY.
   - Для свойств добавлены SORT.
   - В extraBasketFields добавлены/уточнены поля:
     STORE_ID, CATALOG_STORE_ID, MARKDOWN_STORE_ID, MARKDOWN_ROW_KEY, MARKDOWN_CATEGORY.

3) В файле логики условия акции condition_warehouse.php внесены изменения:
   - Файл: /local/php_interface/include/sale_discount/condition_warehouse.php
   - Добавлено извлечение склада из данных конкретной строки корзины:
     MARKDOWN_STORE_ID, STORE_ID, CATALOG_STORE_ID (включая формат через PROPS).
   - Добавлена приоритетная проверка условия акции по складу строки корзины.
   - Оставлен fallback-путь через стандартную проверку привязки товара к складу.

Статус по проблемам:
1) Проблема с ценами акций остаётся актуальной.
   - В модальном окне всё ещё может подтягиваться некорректная скидка по строкам markdown.
   - Нужна дополнительная доработка в логике применения маркетинговых акций.

2) Проблема с добавлением только одного markdown-товара в корзину остаётся актуальной.
   - При попытке добавить две позиции одного товара с разных складов в корзине остаётся только одна строка.
   - Вероятная причина: объединение строк внутри обработчика addProductToBasket модуля b2bcabinet (на стороне контроллера корзины).

Итог:
Выполненные изменения внесены, но две ключевые бизнес-проблемы (акции и раздельное добавление двух markdown-позиций) полностью не решены и требуют дальнейшей доработки.
